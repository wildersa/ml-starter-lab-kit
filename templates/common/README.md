# Common Templates

Reusable templates for optional modules generated by `create_ml_starter.py`.

These templates are intentionally lightweight and avoid external dependencies. They are meant to give a clean starting point, not a complete framework.

## Templates

| Template | Generated module | Purpose |
|---|---|---|
| `eda.py.tpl` | `eda.py` | basic dataset inspection helpers |
| `preprocessing.py.tpl` | `preprocessing.py` | missing values, type conversion, simple scaling helpers |
| `visualization.py.tpl` | `visualization.py` | placeholder chart helpers, no plotting dependency by default |
| `metrics.py.tpl` | `metrics.py` | simple metrics implemented with standard library |
| `optimization.py.tpl` | `optimization.py` | simple grid/random search scaffolding |
| `feature_measurement.py.tpl` | `feature_measurement.py` | compare feature impact and document keep/remove decisions |
| `notebook_factory.py.tpl` | `notebook_factory.py` | create starter `.ipynb` files using JSON |

## Dependency rule

Keep templates standard-library first.

If a project needs pandas, scikit-learn, matplotlib, xgboost, TensorFlow, Keras, PyTorch, or another library, add it inside that generated project only when needed.

## Placeholders

Templates can use these placeholders:

```text
{{PROJECT_NAME}}
{{PACKAGE_NAME}}
{{TASK}}
```
