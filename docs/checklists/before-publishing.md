# Before Publishing

Use this checklist before sharing a notebook, report, or repository.

- [ ] README explains the problem and how to run the project.
- [ ] No secrets, tokens, or private data are committed.
- [ ] Dataset source and license are documented.
- [ ] Large artifacts are not accidentally versioned.
- [ ] Model limitations are documented.
- [ ] Metrics are explained.
- [ ] Generated reports are stored under `reports/`.
- [ ] Notebooks are readable from top to bottom.
- [ ] Reusable logic is moved to `src/`.
- [ ] The project can be understood without oral explanation.
