# Análise Exploratória de Dados

EDA é o primeiro olhar sobre o dataset.

O objetivo é entender problemas antes do treino.

## Checagens básicas

- número de linhas;
- número de colunas;
- tipos das colunas;
- valores ausentes;
- linhas duplicadas;
- distribuição do target;
- intervalo de datas;
- colunas suspeitas;
- possível vazamento.

## Boa saída de EDA

Uma boa EDA deve responder:

```text
Esse dataset sustenta o modelo que quero treinar?
```

## Aviso prático

Não fique preso para sempre na EDA. Encontre o suficiente para criar o primeiro baseline confiável.
