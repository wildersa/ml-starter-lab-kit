# Exploratory Data Analysis

EDA is the first look at the dataset.

The goal is to understand problems before training.

## Basic checks

- number of rows;
- number of columns;
- column types;
- missing values;
- duplicated rows;
- target distribution;
- date range;
- suspicious columns;
- possible leakage.

## Good EDA output

A good EDA should answer:

```text
Can this dataset support the model I want to train?
```

## Practical warning

Do not spend forever on EDA. Find enough to make the first reliable baseline.
